# 如何記錄資料處理（簡易版）

## 為什麼要記錄資料處理？

想像一下：
- 你處理了資料，AI 結果變好了
- 但過了一個月，你忘記當初怎麼處理的...

所以我們要記錄每次的資料處理方式！

---

## 資料版本是什麼？

就像遊戲的「存檔」一樣：

| 版本 | 說明 |
|------|------|
| raw（原始） | 最原始的照片，**絕對不能改** |
| v1.0_baseline | 第一版處理 |
| v1.1_增強 | 在 v1.0 基礎上加了東西 |
| v2.0_清理 | 大改動，刪掉一些不好的照片 |

### 版本命名規則

```
v主版本.次版本_簡短說明

範例：
v1.0_baseline    → 第一版基礎處理
v1.1_augmented   → 小改動：加了資料增強
v2.0_cleaned     → 大改動：刪除模糊照片
```

---

## 常見的資料處理方式

### 1. 資料分割

把 840 張照片分成：
- **訓練集**：給 AI 學習用（例如 80%）
- **驗證集**：檢查 AI 學得好不好（例如 20%）

```
原始 840 張
    ↓ 分割
訓練集 672 張 (80%)
驗證集 168 張 (20%)
```

### 2. 資料增強

把現有的照片「變形」產生更多照片：

| 方法 | 說明 |
|------|------|
| 水平翻轉 | 照片左右對調 |
| 旋轉 | 照片轉個角度 |
| 縮放 | 照片放大縮小 |
| 調整亮度 | 照片調亮或調暗 |

這樣可以讓 AI 看更多種類的照片，學得更好。

### 3. 處理不平衡

因為「會著床」的照片太少，可以：
- **過採樣**：把少的那類複製多份
- **欠採樣**：把多的那類刪掉一些

## 範例：v1.0_baseline 資料處理記錄

### 基本資訊

- **版本名稱**：v1.0_baseline
- **建立日期**：2025-01-25
- **建立者**：王小明

### 做了什麼處理

1. **分割資料**
   - 訓練集：672 張（80%）
   - 驗證集：168 張（20%）
   - 用「分層抽樣」確保兩邊的比例一樣

2. **沒有做資料增強**（這是基準版本）

### 處理後的資料

| 資料集 | 不會著床 (0) | 會著床 (1) | 總數 |
|--------|-------------|-----------|------|
| 訓練集 | 573 張 | 99 張 | 672 張 |
| 驗證集 | 143 張 | 25 張 | 168 張 |

### 備註

這是最基本的處理，之後會嘗試加入資料增強。

## 重要提醒

### ⚠️ 絕對不要改原始資料！

原始照片放在 `data/raw/` 資料夾。
**這個資料夾裡的東西絕對不能改、不能刪！**

所有處理後的資料都放在 `data/processed/` 資料夾。

### 資料和實驗的關係

```
資料版本 v1.0 ←── 用在 ──→ 實驗 EXP-001, EXP-002

資料版本 v1.1 ←── 用在 ──→ 實驗 EXP-003, EXP-004
```

這樣如果某個實驗結果很好，我們可以追溯回去看用的是哪個版本的資料。
